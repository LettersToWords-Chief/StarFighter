# Session Start Workflow

## When to run
- At the beginning of any new conversation about Star Fighter
- Any time during a session when context may have been lost (e.g., after a long gap, or when the user references a plan or feature that isn't in active memory)
- When in doubt, re-read the brief rather than assume

## Step 1: Read the project brief
Use `view_file` to read:
```
c:\Star Fighter\PROJECT_BRIEF.md
```

## Step 2: Orient
From the brief:
- Note what is ✅ Done vs 📋 Planned
- Note the most recent work ("In Progress / Most Recent Work" section)
- Recall the design principles (Section D)

## Step 3: Ask, don't assume
Ask Bill what he wants to work on today. Do not propose changes or begin implementation until asked.

## Important rules
- Do not rely on conversation history alone for project state — the context window is finite and plans written earlier in a long session can be lost
- The PROJECT_BRIEF.md is the authoritative source of truth for project state
- After any significant plan is approved or feature is completed, update PROJECT_BRIEF.md to reflect the new state
